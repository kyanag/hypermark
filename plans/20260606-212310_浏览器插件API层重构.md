# 浏览器插件 API 层重构计划

> 生成时间: 2026-06-06 21:23:10

## 一、背景

现有插件的 API 调用分散在多个文件中，且使用的接口路径与新 API 文档不匹配。

### 当前问题

| 文件 | 问题 |
|------|------|
| `src/shared/api.ts` | 单一 `HyperMarkApi` 类，所有方法平铺；接口路径为旧版（如 `/mark`、`/unmark`、`/is_marked`） |
| `src/content/content.ts` | 内联 `fetchApi` + `doBookmark`/`doUnbookmark`/`loadCategories`，调用 `/update_mark`、`/add_category` 等**服务端不存在**的接口 |
| `src/background/background.ts` | 通过 `chrome.runtime.onMessage` 转发调用，message type 与 api.ts 方法耦合 |

### 新旧接口对照（与插件相关的部分）

| 旧接口（现有代码） | 新接口（API 文档） | 变化 |
|---|---|---|
| `POST /mark` | `POST /api/mark` | 路径前缀 |
| `POST /unmark` | `POST /api/unmark` | 路径前缀 |
| `GET /is_marked` | `GET /api/is_marked` | 路径前缀 |
| `GET /categories` | `GET /api/categories` | 路径前缀 |
| `POST /add_category` | `POST /api/categories` | **端点完全不同** |
| `POST /update_mark` | `PUT /api/links/{urlEncoded}` | **端点完全不同**，需要 URL 编码 |
| `POST /add_tag` | `POST /api/links/{urlEncoded}/tags` | **端点完全不同** |
| `POST /move_category` | `PUT /api/links/{url}`（部分更新） | **接口语义改变** |
| `POST /add_domain_cname` | `POST /api/domain-cnames` | 路径改变 |
| `GET /links` | `GET /api/links?site=&category=&tag=&limit=&offset=` | 查询参数改变 |
| `GET /health` | `GET /api/health` | 路径前缀 |
| 无 | `GET /api/parse?url=xxx` | **新增**，面板数据加载使用 |
| 无 | `GET /api/tags/autocomplete?q=xxx` | **新增**，标签自动补全 |

## 二、目标

将 API 请求层按**功能**而非接口进行模块化，放入 `src/core/` 目录：

```
src/core/
├── types.ts           # 类型定义（与服务端数据模型对齐）
├── http.ts            # HTTP 客户端（基础请求、Method Override、错误处理）
├── services.ts        # 业务服务（收藏状态检查、快速收藏、取消收藏、面板加载/保存/删除）
└── index.ts           # 统一导出
```

### 功能模块划分

| 模块 | 功能 | 涉及的 API 端点 |
|------|------|---|
| **services** | 页面收藏状态检查 | `GET /api/is_marked` |
| | 页面快速收藏 | `POST /api/mark` |
| | 页面取消收藏 | `POST /api/unmark` |
| | 面板数据加载（URL 解析 + 辅助数据） | `GET /api/parse` → `GET /api/categories` → `GET /api/tags/autocomplete` |
| | 面板数据保存（新建/更新 + 分类创建 + 标签操作） | `POST /api/links` / `PUT /api/links/{urlEncoded}` / `POST /api/categories` / `POST /api/links/{urlEncoded}/tags` |
| | 面板数据删除 | `POST /api/unmark` |

### 关键设计原则

1. **功能聚合**：面板保存时，自动组合多个 API 调用（创建分类 → 保存链接 → 添加标签），调用方只需调一个方法
2. **错误透传**：每个功能方法返回结构化结果 `{ success: boolean; data?: T; error?: string }`，由 UI 层决定提示
3. **Method Override**：所有 DELETE/PATCH/PUT 请求通过 `POST + X-HTTP-Method-Override` 发送，兼容浏览器插件环境
4. **类型与服务端对齐**：`types.ts` 的接口严格对应 API 文档的数据模型

## 三、详细设计

### 3.1 `src/core/types.ts` — 类型定义

```typescript
// 服务端 Link 模型
export interface Link {
  url: string;
  title: string;
  createdAt: string;      // ISO8601
  category: string | null;
  tags: string[];
  page: Page | null;
}

// 服务端 Page 模型
export interface Page {
  hyperId: string;
  site: string;
  route: string;
  std: string;
  args: Record<string, string>;
  tags: string[];
  createAt: string;
}

// 服务端 Category 模型
export interface Category {
  name: string;
  parentId: string | null;
  createdAt: string;
}

// 服务端 Tag 模型
export interface Tag {
  id: number;
  name: string;
  title: string;
  createdAt: string;
}

// 服务端 Site 模型（简要）
export interface SiteBrief {
  name: string;
  title: string;
}

// 收藏状态检查结果
export interface MarkStatus {
  marked: boolean;
  link?: Link;            // 已收藏时附带
}

// URL 解析结果
export interface ParseResult {
  page: Page;
  link: Link | null;      // null 表示未收藏
}

// 链接列表分页结果
export interface LinkListResponse {
  data: Link[];
  total: number;
  limit: number;
  offset: number;
}

// 统一操作结果
export interface ActionResult<T = void> {
  success: boolean;
  data?: T;
  error?: string;
}
```

### 3.2 `src/core/http.ts` — HTTP 客户端

```typescript
// 核心职责：
// 1. 基础 URL 管理（从 chrome.storage.local 读取，可动态修改）
// 2. 统一请求封装：自动添加 Content-Type，处理状态码
// 3. Method Override：DELETE/PATCH/PUT 通过 POST + header 伪装
// 4. 超时控制：5s 超时，超时视为服务不可用
// 5. 错误解析：从响应 body 提取 error 字段

export class HttpClient {
  private _baseUrl: string;

  constructor(baseUrl: string);
  get baseUrl(): string;
  setBaseUrl(url: string): void;

  // 内部使用，上层模块不直接调用
  private request<T>(method: string, path: string, body?: object): Promise<T>;

  // 快捷方法
  get<T>(path: string): Promise<T>;
  post<T>(path: string, body?: object): Promise<T>;
  put<T>(path: string, body?: object): Promise<T>;    // 自动 Method Override
  delete(path: string): Promise<void>;                  // 自动 Method Override
  patch<T>(path: string, body?: object): Promise<T>;   // 自动 Method Override
}
```

**Method Override 逻辑**：
- `put()` / `delete()` / `patch()` 内部发送 `POST` 请求，添加 `X-HTTP-Method-Override` 头
- `get()` / `post()` 直接发送

### 3.3 `src/core/services.ts` — 业务服务

单一 `HyperMarkService` 类，涵盖页面收藏 + 面板数据两类场景：

```typescript
// 核心职责：页面嵌入按钮和面板中的所有 API 交互

export class HyperMarkService {
  constructor(private http: HttpClient);

  // ===== 页面收藏场景 =====

  // 1. 检查当前 URL 是否已收藏
  async checkMarked(url: string): Promise<ActionResult<MarkStatus>>;

  // 2. 页面快速收藏（按钮一键收藏，不附带分类和标签）
  async quickAdd(url: string, title?: string): Promise<ActionResult>;

  // 3. 页面取消收藏（按钮操作）
  async remove(url: string): Promise<ActionResult>;

  // ===== 面板数据场景 =====

  // 4. 面板数据加载
  //    组合调用：GET /api/parse?url=xxx → GET /api/categories
  //    返回解析后的 Page 信息 + 已收藏的 Link 信息 + 分类列表
  async loadPanelData(url: string): Promise<ActionResult<PanelLoadResult>>;

  // 5. 面板数据保存（面板确认）
  //    自动处理：新建分类(如有) → 新建/更新链接 → 标签增删
  //    详见下方 save() 流程说明
  async save(saveData: PanelSaveData): Promise<ActionResult<Link>>;

  // 6. 面板数据删除
  async deleteFromPanel(url: string): Promise<ActionResult>;

  // 辅助方法
  async searchTags(query: string, limit?: number): Promise<ActionResult<Tag[]>>;
  async createCategory(name: string): Promise<ActionResult<Category>>;
}

// 面板加载结果
export interface PanelLoadResult {
  parseResult: ParseResult;
  categories: Category[];
}

// 面板保存数据
export interface PanelSaveData {
  url: string;
  title: string;
  category?: string;           // 分类名称
  newCategory?: string;        // 新增分类名称
  tagsToAdd?: string[];        // 要添加的标签名称
  tagsToRemove?: number[];     // 要移除的标签 ID
  bookmarked: boolean;         // 当前是否已收藏
}
```

**save() 方法详细流程**：
```
1. 如果 newCategory 有值：
   a. 调用 createCategory(newCategory)
   b. 失败 → 返回错误，不继续
   c. 成功后将 category 设置为 newCategory

2. 如果 bookmarked === true（更新）：
   a. PUT /api/links/{urlEncoded}，body: { title, category }
   b. 失败 → 返回错误

3. 如果 bookmarked === false（新建）：
   a. POST /api/links，body: { url, title, category }
   b. 失败 → 返回错误

4. 如果有 tagsToAdd：
   a. POST /api/links/{urlEncoded}/tags，body: { tags: tagsToAdd }
   b. 失败 → 记录警告但不阻断（非致命错误）

5. 如果有 tagsToRemove：
   a. 逐个 DELETE /api/links/{urlEncoded}/tags/{tagId}
   b. 失败 → 记录警告但不阻断

6. 返回最终 Link 对象
```

## 五、实施步骤

### Step 1: 创建 `src/core/types.ts`
- 定义所有服务端数据模型
- 定义 `ActionResult<T>` 统一返回格式

### Step 2: 创建 `src/core/http.ts`
- 实现 `HttpClient` 类
- 实现 Method Override（POST + X-HTTP-Method-Override）
- 实现超时、错误处理

### Step 3: 创建 `src/core/services.ts`
- 实现 `HyperMarkService` 类，包含所有业务方法
- 收藏场景：`checkMarked()` / `quickAdd()` / `remove()`
- 面板场景：`loadPanelData()` / `save()` / `deleteFromPanel()` / `searchTags()` / `createCategory()`
- `save()` 内部组合调用：创建分类 → 新建/更新链接 → 标签操作

### Step 4: 创建 `src/core/index.ts`
- 统一导出 + `createService()` 工厂

### Step 5: 修改 `src/background/background.ts`
- 导入 `createService()`
- 重写所有 `onMessage` 处理器，使用新 service 方法

### Step 6: 修改 `src/content/content.ts`
- 导入 `createService()` 替换内联 API 调用
- `checkBookmarked()` → `service.checkMarked()`

- `doUpdateMark()` → `service.save()`
- 新增分类逻辑由 `service.save()` 内部处理，移除内联 `/add_category` 调用

### Step 7: 清理 `src/shared/api.ts`
- 标记废弃注释，或直接删除

## 六、风险点

| 风险 | 说明 | 应对 |
|------|------|------|
| 旧版接口兼容 | 服务端 `/mark`、`/unmark`、`/is_marked` 仍可用但路径不同 | 统一使用 `/api/` 前缀 |
| Method Override | 服务端使用 `UseHttpMethodOverride()` 中间件 | http.ts 中自动添加 header |
| 标签操作失败阻断 | `service.save()` 中标签操作失败不应阻断主流程 | 标签操作错误作为警告返回，不 throw |
| URL 编码 | `PUT /api/links/{urlEncoded}` 路径参数需要 `encodeURIComponent()` | http.ts 或 services.ts 内部处理 |
| 分类 ID vs 名称 | 旧代码使用 `categoryId`（number），新 API 使用分类名称（string） | 所有分类操作统一使用名称 |
